# Portfolio Starter Template

This is a website starter template based on the structure of my deployed portfolio website, but without my personal content, images, resume files, game, backend server, or original styling.

It is meant for someone who is new to coding and wants a simple portfolio they can edit one file at a time.

## Quick Start

1. Open `index.html` in a browser.
2. Edit `content.js`.
3. Replace the sample name, links, timeline, skills, projects, and contact info.
4. Add personal images or a resume under `assets/`.
5. Refresh the browser to see changes.

No install step is required.

## What To Edit

- `content.js`: Most personal content lives here. Start with this file.
- `styles.css`: Colors, spacing, typography, and layout.
- `index.html`: Page structure. Most beginners should only make small changes here.
- `script.js`: Renders the content and handles filters, theme toggle, and mobile navigation.
- `assets/`: Put a resume, profile photo, and project screenshots here.

## Publishing Options

The cleanest option is to copy the contents of this folder into a new repo and publish that repo with GitHub Pages. That keeps the new site independent from my portfolio.

## Beginner Checklist

- Change `name`, `initials`, `title`, `location`, and `email`.
- Replace all placeholder links.
- Add a real resume PDF or hide the resume button by setting `resumeUrl: ""`.
- Add at least one real project screenshot.
- Keep project summaries short: problem, what you built, result.
- Test on a phone-sized browser window before publishing.
